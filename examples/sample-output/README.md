# Sample output — what the DevLoop chain produces

These files illustrate the **shape** of DevLoop's artifacts for a small feature
("self-service partner SFTP onboarding"), grounded in the bundled
[`../integrations-src/`](../integrations-src/) sources (SSO, Transactional Email, SFTP):

- [`requirements.md`](requirements.md) — numbered FR/NFR, each tagged with the wiki concept +
  source it traces to (the Requirements Analyst's output).
- [`stories.md`](stories.md) — epics → INVEST user stories → Gherkin acceptance criteria +
  a traceability matrix (the Story Writer's output).

> **Illustrative, hand-authored** to show the format and the grounding/traceability style — not
> a captured model run. Your real artifacts are generated by the agent against *your* wikis, so
> the wording and depth will differ. They follow the same templates in `core/shared/`.
